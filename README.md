# AWS Well-Architected Checklist

This repository is a practical reference based on the AWS Well-Architected Framework.  
It organizes key review questions, common issues, and improvement approaches across all five pillars.

## Purpose
The goal of this repository is to build structured architectural thinking by applying Well-Architected principles to real-world AWS environments, especially in pre-sales and solution review scenarios.

## What This Repository Covers
- Architecture review perspectives across all five pillars
- Common gaps seen in AWS accounts
- Trade-offs between cost, performance, availability, and security
- Practical improvement approaches rather than theory

## Pillars Included
- Operational Excellence
- Security
- Reliability
- Performance Efficiency
- Cost Optimization

Each pillar is documented separately under the `pillars/` directory for clarity and easy reference.

## Who This Is For
- Cloud practitioners and AWS learners
- Pre-sales and solution architecture roles
- Anyone preparing for AWS Solution Architect responsibilities

## Disclaimer
This content is based on learning, hands-on exposure, and publicly available AWS best practices.  
It is intended for educational and reference purposes.
